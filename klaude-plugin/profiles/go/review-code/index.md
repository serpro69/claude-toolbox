# Go — review checklists

Consumed by the `review-code` skill. When the `go` profile is active, every checklist in **Always load** is applied to the diff. There are no conditional entries for this profile.

## Always load

- [security-checklist.md](security-checklist.md) — Go-specific security smells: input validation, crypto misuse, secret handling, context cancellation, common Go-ecosystem vulnerabilities.
- [solid-checklist.md](solid-checklist.md) — SOLID design-principle smells expressed in Go terms: package cohesion, interface segregation, dependency inversion, composition over inheritance.
- [code-quality-checklist.md](code-quality-checklist.md) — idiomatic Go quality signals: error handling, naming, goroutine discipline, defer correctness, maintainability.
- [removal-plan.md](removal-plan.md) — staged-removal template for Go code being retired (exports, packages, modules).
